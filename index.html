Bakong Mini Service Integration Guide

Complete guide for integrating your web service with the Bakong mobile app. This document covers all required bridge methods, event handling, and implementation best practices.

Required bridge methods must work correctly for your service to be accepted. Conditional methods become required when you show the toolbar. Optional methods enhance user experience but are not mandatory.

Bridge Method Requirements

paymentInput() REQUIRED
goBack() REQUIRED
dismiss() REQUIRED
setTitle() CONDITIONAL
setStatusBarColor() CONDITIONAL
orderCompleted() OPTIONAL
setToolbarVisible() OPTIONAL
setStatusBarIconColor() OPTIONAL

üìã Conditional Requirements

  • setTitle() - To set a meaningful title
  • setStatusBarColor() - To ensure proper status bar appearance

Payment Integration Flow

1. Initialize Payment

Call paymentInput() with payment data when user wants to make a payment.

2. Handle Payment Completion

Listen for PaymentCompleted event from mobile when payment succeeds.

3. Complete Order (Optional)

Call orderCompleted() to notify mobile that the order is complete.

Required Implementation

1. Payment Input

// Always check if bridge is available before calling
if (window.Bakong && typeof window.Bakong.paymentInput === 'function') {
    const paymentData = {
        amount: "54000",
        currencyCode: "116",
        currencyName: "KHR",
        description: "Book tickets",
        merchantName: "Your Service Name",
        receiverAccountId: "your-account@devb",
        receiverName: "Your Account Name",
        orderId: "ORDER_123456"
    };
    window.Bakong.paymentInput(JSON.stringify(paymentData));
} else {
    console.error('Bakong bridge not available');
}

2. Payment Completion Event Handling

// Listen for payment completion events from mobile
window.addEventListener('PaymentCompleted', function(event) {
    try {
        const paymentData = JSON.parse(event.detail);
        console.log('Payment completed:', paymentData);
        
        // Handle successful payment
        const { orderId, transactionHash, transactionId } = paymentData;
        
        // Update your UI, show success message, etc.
        showPaymentSuccess(orderId, transactionHash);
        
        // Optionally notify mobile that order is complete
        if (window.Bakong && typeof window.Bakong.orderCompleted === 'function') {
            window.Bakong.orderCompleted(JSON.stringify({
                orderId: orderId,
                transactionId: transactionId || transactionHash
            }));
        }
    } catch (e) {
        console.error('Error parsing payment completion data:', e);
    }
});

3. Navigation Methods

// Go back functionality
function goBack() {
    if (window.Bakong && typeof window.Bakong.goBack === 'function') {
        window.Bakong.goBack();
    }
}

// Close/dismiss service
function closeService() {
    if (window.Bakong && typeof window.Bakong.dismiss === 'function') {
        window.Bakong.dismiss();
    }
}

Appearance Control (When Using Toolbar)

1. Show Toolbar and Set Title

// Show toolbar and set title (both become required)
if (window.Bakong && typeof window.Bakong.setToolbarVisible === 'function') {
    window.Bakong.setToolbarVisible(true);
}

if (window.Bakong && typeof window.Bakong.setTitle === 'function') {
    window.Bakong.setTitle('Your Service Page');
}

2. Set Status Bar and Toolbar Color

// Set status bar and toolbar color (both will match)
if (window.Bakong && typeof window.Bakong.setStatusBarColor === 'function') {
    window.Bakong.setStatusBarColor('#121212'); // Dark theme
    // or
    window.Bakong.setStatusBarColor('#FFFFFF'); // Light theme
}

3. Set Status Bar Icon Color

// Set status bar icon color
if (window.Bakong && typeof window.Bakong.setStatusBarIconColor === 'function') {
    window.Bakong.setStatusBarIconColor(true);  // Light icons (for dark background)
    // or
    window.Bakong.setStatusBarIconColor(false); // Dark icons (for light background)
}

Mobile-to-Web Events

1. PaymentCompleted Event

Mobile sends this event when paymentInput() succeeds:

window.dispatchEvent(new CustomEvent('PaymentCompleted', { 
    detail: "{\"orderId\":\"TR2509-101504183252850826\",\"transactionHash\":\"6aab62b37465ae4901f621a30af08f08b446eee1b1034428299eddc953a55417\",\"transactionId\":\"\"}" 
}));

2. CloseRequest Event

Mobile sends this event when user clicks close button:

(function() {
    // Dispatch close request event to web
    var event = new CustomEvent('CloseRequest', { 
        detail: { source: 'bakong' }
    });
    window.dispatchEvent(event);
    console.log('Close request sent to web');
})();

3. Handle Close Request

// Listen for close request events from mobile
window.addEventListener('CloseRequest', function(event) {
    console.log('Close request received from mobile');
    
    // Show confirmation dialog
    const confirmed = confirm('Do you want to close this service?');
    
    if (confirmed) {
        // Call dismiss() to close the service
        if (window.Bakong && typeof window.Bakong.dismiss === 'function') {
            window.Bakong.dismiss();
        }
    }
});

Best Practices

1. Always Check Bridge Availability

// Good practice
if (window.Bakong && typeof window.Bakong.methodName === 'function') {
    window.Bakong.methodName();
} else {
    console.error('Bridge method not available');
}

2. Handle Errors Gracefully

try {
    const paymentData = JSON.parse(jsonString);
    // Process payment data
} catch (e) {
    console.error('Invalid payment data:', e);
    // Show error to user
}

3. Validate Required Fields

function validatePaymentData(data) {
    const required = ['amount', 'currencyCode', 'receiverAccountId', 'merchantName'];
    const missing = required.filter(field => !data[field]);
    
    if (missing.length > 0) {
        throw new Error(`Missing required fields: ${missing.join(', ')}`);
    }
    
    return true;
}

4. Use Proper User-Agent

Your service will receive requests with the Bakong User-Agent format:

Bakong:1.0.0:EN:Android:username@devb:phone:email:displayname

Common Issues & Solutions

Issue: Bridge methods not available

Solution: Always check window.Bakong exists before calling methods.

Issue: Payment data validation fails

Solution: Ensure all required fields are present and properly formatted.

Issue: Status bar colors not working

Solution: Use valid hex colors (e.g., #121212) and call setStatusBarColor() after showing toolbar.

Issue: Events not received

Solution: Add event listeners before calling bridge methods that trigger events.
